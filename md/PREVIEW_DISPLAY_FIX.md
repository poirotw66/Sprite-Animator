# Preview Display Fix - 預覽顯示修正

## 🐛 問題描述 (Problem Description)

### 現象
- **前端預覽**: 人物的一些特徵色彩(如粉紅色衣服、綠色配件)會被誤判為背景色,導致顯示異常
- **下載的 PNG**: 去背完全正常,沒有問題

### 根本原因
問題出在 `utils/imageUtils.ts` 中的 `isChromaKeyPixel()` 函數。這個函數用於**內容分析**(計算邊界框和中心點),而不是實際的去背處理。

**實際去背**是在 `workers/chromaKeyWorker.ts` 中完成的,所以下載的 PNG 是正常的。

**前端顯示問題**是因為 `isChromaKeyPixel()` 的檢測邏輯太寬鬆,導致:
1. 人物的粉紅色/綠色被誤判為背景色
2. 邊界框計算錯誤
3. 中心點計算錯誤
4. 顯示時人物被錯誤裁剪或位置偏移

## ✅ 解決方案 (Solution)

### 修正 `isChromaKeyPixel()` 函數

**檔案**: `utils/imageUtils.ts`

#### 之前的邏輯 (太寬鬆)
```typescript
const isMagentaLike = (r > 180 && g < 100 && b > 100) ||
  (r > 150 && g < 80 && b > 150) ||
  (r > 200 && g < 50 && b > 200);
const isGreenLike = (g > 180 && r < 100 && b < 100) ||
  (g > 150 && r < 80 && b < 80);
```

**問題**: 
- 粉紅色衣服 (例如 R=200, G=80, B=150) 會被誤判為洋紅色
- 綠色配件 (例如 G=180, R=50, B=50) 會被誤判為綠幕

#### 修正後的邏輯 (嚴格)
```typescript
// Pure magenta chroma key (#FF00FF): R=255, G=0, B=255
// Only match if it's VERY close to pure magenta
const isPureMagenta = r > 240 && g < 30 && b > 240 && Math.abs(r - b) < 20;

// Target green: #00FF00 (R=0, G=255, B=0). Also match #00B140-like for compatibility.
const isPureGreen = g > 240 && r < 30 && b < 30;
const isStandardGreen = g > 150 && g < 200 && r < 30 && b < 100 && 
  (g - r) > 120 && (g - b) > 50;
```

**改進**:
- ✅ 只匹配**非常接近**純色度去背色的像素
- ✅ 粉紅色衣服不會被誤判 (需要 R>240, G<30, B>240)
- ✅ 綠色配件不會被誤判 (需要 G>150, R<30, B<100, 且 G-R>120)
- ✅ 增加 `Math.abs(r - b) < 20` 確保洋紅色是純色

## 🔍 技術細節 (Technical Details)

### `isChromaKeyPixel()` 的用途

這個函數**只用於內容分析**,在 `analyzeFrameContent()` 函數中:

1. **計算邊界框** (Bounding Box)
   - 找出人物的最小/最大 X/Y 座標
   - 用於自動裁剪和定位

2. **計算中心點** (Center of Mass)
   - 計算人物的重心位置
   - 用於對齊和居中

3. **計算核心區域** (Core Region)
   - 分析軀幹區域(更穩定)
   - 用於更精確的定位

### 為什麼不影響下載的 PNG?

因為**實際的去背處理**是在 `workers/chromaKeyWorker.ts` 中完成的,使用完全不同的、更精確的算法。`isChromaKeyPixel()` 只影響前端顯示時的內容分析。

## ⚠️ 注意事項 (Notes)

### 這個函數的設計原則

1. **只匹配真正的色度去背色**
   - 必須非常接近純色 (#FF00FF 或 #00FF00)
   - 不應該匹配人物的任何顏色

2. **寧可漏判,不可誤判**
   - 如果邊界框稍微大一點(包含一些背景),沒關係
   - 但如果誤判人物顏色,會導致嚴重問題

3. **只影響顯示,不影響去背**
   - 實際的去背由 Worker 完成
   - 這個函數只影響內容分析
